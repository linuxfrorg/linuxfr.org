Gagnants et prix ¶

Vous pouvez gagner des livres des éditions Eyrolles, ENI ou InLibroVeritas, ou encore des abonnements Linux Mag France en contribuant au site : soumission de dépêche, journaux, amélioration du code du site, logos, feuilles de style, etc.

Visiteur (non) authentifié, ou pourquoi ouvrir un compte ? ¶

Pour passer du statut de visiteur non authentifié à celui de visiteur authentifié, il suffit de se créer un compte sur le site et de se connecter avec.

Un visiteur non authentifié peut :

  • lire le site (dépêches, journaux, forums, sondages, tâches du système de suivi, les commentaires associés, flux RSS) ;
  • proposer des dépêches et des bogues/suggestions dans le système de suivi.

Un visiteur authentifié peut :

  • écrire des commentaires sur les différentes ressources du site (dépêches, journaux, forums, sondages, tâches du système de suivi, tribunes) ;
  • activer la barre de navigation du site pour se déplacer plus rapidement dans les nouveaux commentaires ou pour filtrer les commentaires mal notés ;
  • changer l'apparence du site (choix de la CSS) ;
  • fournir son adresse de courriel pour être contacté ;
  • noter les commentaires des autres visiteurs ;
  • voir les commentaires nouveaux depuis sa dernière visite ;
  • retrouver ses dernières contributions dans sa page personnelle ;
  • écrire dans les forums ou publier des journaux ;
  • voter pour le bogue/la suggestion la plus prioritaire dans le système de suivi ;
  • être remarqué pour la qualité de ses contributions et devenir relecteur ou modérateur.

Barre d'outils ¶

La barre d'outils (toolbar) du site n'est que pour les visiteurs authentifiés.

Elle permet :

  • d'adapter le seuil de visualisation des commentaires (à -42 vous aurez tout, même les commentaires les plus mal notés)
  • d'enrouler / dérouler un commentaire avec une élégante petite case [-] ou [+]. C'est notamment utile pour les commentaires masqués à cause du seuil de visualisation, pour éviter d'avoir à ouvrir un nouvel onglet/une nouvelle page pour tout de même lire le commentaire
  • de bénéficier d'une navigation rapide à la souris, en cliquant sur < # >, de nouveau commentaire en nouveau commentaire, pour ne pas user la molette de votre souris
  • de bénéficier d'une navigation rapide au clavier, avec les touches < et > (j et k fonctionnent aussi), de nouveau commentaire en nouveau commentaire, et pour les nouveaux commentaires sur les contenus [ et ] (h et l fonctionnent aussi), pour ne pas lâcher votre clavier des mains pour utiliser une souris
  • de retourner en haut de la page en appuyant sur g ou en bas de la page avec G

Boîte d'identification personnelle ¶

Une boîte d'identification personnelle apparaît sur chaque page sous le logo. Elle contient différentes informations :

  • Votre login qui pointe vers l'adresse de votre page personnelle LinuxFr.org ;
  • Une ligne qui vous donne le nombre de fois où vous pouvez encore donner votre avis sur un commentaire aujourd'hui ainsi que le total quotidien. Si vous n'avez pas cette ligne c'est que vous n'avez pas la possibilité de le faire ;
  • Différents liens pour la gestion de votre compte ;

Note : Le nombre d'avis auquel vous avez droit peut évoluer.

Notes sur les contenus ¶

Les contenus du site (dépêches, journaux, forums, sondages, système de suivi, propositions d'entretien, etc.) peuvent être notés pertinent (+1) ou inutile (-1).

Il n'est pas possible de noter ses propres contenus, de noter un même contenu plusieurs fois ou de noter les contenus anciens. Il faut être authentifié pour pouvoir noter un contenu. L'affichage de la nouvelle note peut être légèrement différée pour des raisons techniques (systèmes de cache en place sur le site).

La note globale d'un contenu est la somme des notes pertinent (+1) et inutile (-1) mises par les visiteurs. Elle peut être positive, négative ou nulle.

L'intérêt d'un contenu est un indicateur calculé à partir de la date de création et de la note globale du contenu. Il sert à trier les contenus récents et bien notés (donc soit les plus récents, soit les suffisamment bien notés et pas trop anciens).

intérêt = date de création + note * coefficient de contenu * coefficient de fréquence

  • date de création en secondes depuis Epoch
  • coefficient de contenu =
    • dépêches : 3
    • forums : 1
    • journaux : 2
    • sondages : 7
    • système de suivi : 2
    • propositions d'entretiens : 7
  • coefficient de fréquence : 864 (~ nombre de secondes par jour / nombre de votes par jour)

Fermer son compte ¶

La « fermeture du compte » consiste en une invalidation du compte, qui devient inutilisable. Pour cela, suivez les instructions sur la page de fermeture de compte. Les contenus et commentaires associés au compte restent inchangés. Si vous ne souhaitez pas que les éventuels nom et prénom renseignés dans les préférences ne continuent d'apparaître en ligne (et soient donc indexés par les moteurs de recherche), pensez à les supprimer avant de fermer le compte (le changement sera effectif à la prochaine suppression du cache temporaire LinuxFr.org).

Si vous souhaitez ne plus faire apparaître votre pseudo/login pour les mêmes raisons, envoyez une demande aux administrateurs de fermeture d'un compte avec anonymisation (tous les contenus et commentaires associés seront affectés à Anonyme). Cette dernière opération est définitive.

Il n'est pas possible d'utiliser à nouveau le pseudo/login ou l'adresse de courriel associés à un compte fermé :

  • supprimer l'adresse de courriel d'un compte fermé nécessite de faire une demande aux administrateurs
  • libérer un pseudo/login nécessite une purge du compte et donc une demande aux administrateurs

La « purge du compte » consiste à totalement supprimer le compte et à anonymiser ou supprimer tous les contenus associés. Seuls les administrateurs peuvent réaliser cette opération, définitive par nature.

Certificat SSL/TLS et alertes dans les navigateurs ¶

SSL/TLS est un protocole de sécurisation des échanges sur le web. Lorsque vous accédez à une page du site dont l'adresse commence par https://linuxfr.org (la partie importante est le s de https), votre navigateur web utilise le protocole SSL/TLS pour discuter avec le site LinuxFr.org. SSL/TLS repose sur l'utilisation de certificat, pour qu'un site prouve, autant que possible, qu'il est bien celui qu'il prétend être et pour éviter que l'utilisateur ne soit leurré.

LinuxFr.org utilise un certificat SSL/TLS fourni par CAcert, une autorité de certification basée sur une communauté. Malheureusement, le certificat racine de l'autorité CAcert n'est pas inclus par défaut notamment dans les navigateurs les plus utilisés (dont Mozilla Firefox).

Si vous n'avez pas rajouté le certificat racine de CAcert dans votre navigateur manuellement ou via les dépôts de votre distribution GNU/Linux, il est possible que votre navigateur affiche une alerte de sécurité lors de la visite d'une page https.

Pour que votre navigateur puisse vérifier son authenticité, il doit disposer du certificat racine de l'autorité de certification CAcert. Si ce n'est pas le cas du vôtre, vous devez en premier lieu ajouter le [certificat racine](http://www.cacert.org/index.php?id=3) au trousseau de votre navigateur en cliquant sur le lien 'Root Certificate (PEM Format)', en cochant au moins la case « Ce certificat peut identifier des sites Web. » et en vérifiant l'empreinte numérique SHA1.

Empreinte numérique SHA1 CAcert :
13:5C:EC:36:F4:9C:B8:E9:3B:1A:B2:70:CD:80:88:46:76:CE:8F:33

Une fois ce certificat intégré à votre trousseau, votre navigateur devrait accepter le certificat LinuxFr.org.

Empreinte numérique SHA1 LinuxFr.org :
65:66:48:8F:BE:91:B8:00:52:02:C8:CB:2E:CC:86:5A:37:58:1F:C8

Pourquoi ne prenez pas un certificat SSL/TLS gratuit ou payant de chez Machin qui est par défaut dans un navigateur ? ¶

La gratuité n'est pas le critère principal dans le choix de CAcert. CAcert est une autorité de certification basée sur une communauté (community driven non-Profit Certificate Authority), qui propose à la fois du centralisé et du décentralisé (réseau de confiance), qui publie son code sous licence libre. De plus nous avons eu des échos positifs sur CAcert et nous avons déjà discuté avec certains de leurs admins.

Être reconnue par défaut par les navigateurs est un plus, mais si on se limite à ce critère, on encourage un oligopole d'autorités de confiance, dont certaines ont déjà fait montre de pratiques plus que discutables (liste non exhaustive) :

  • Mozilla Security : Fraudulent *.google.com Certificate [DigiNotar, Iran] (2011)
  • ZdNet : « Des certificats SSL frauduleux de Comodo autorisent des attaques contre les Webmails » (2011)
  • Rue89 : « Tunisie : Microsoft complice de la censure numérique par Ben Ali » (2011)
  • les résultats de l'observatoire SSL de l'Electronic Frontier Foundation présentés à Defcon 2010 et au CCC 2010
  • Page wikipédia Controverses autour de Verisign
  • DLFP : « Verisign attaque l'ICANN » (2004)
  • DLFP : « ICANN vs. Verisign ? » (2003)
  • DLFP : « VeriSign refuse d'obtempérer » (2003)
  • DLFP : « L'ICANN et l'IAB somment Verisign de suspendre leur service de redirection » (2003)
  • DLFP : « VeriSign détruit l'un des fondements d'Internet » (2003)

Système de karma ¶

Le karma est une note interne qui détermine le nombre d'avis journaliers et la possibilité d'écrire sur le site.

L'influence de différentes actions sur le karma :

  • Karma par défaut : 20
  • Note « pertinent » : +1
  • Note « inutile » : -1
  • Dépêche publiée : +50 (pour l'auteur)

Le karma augmente donc lorsque l'on reçoit des notes « pertinent » (quel que soit le type de contenu noté) ou que l'on publie des dépêches.

Le nombre d'avis quotidiens est déterminé à partir du karma. La formule est min(3 + karma / 10, 100). (source).

Le score de base des commentaires dépend du karma. La formule est E( log10( karma ) ) - 1, où E() est la partie entière inférieure si le karma est strictement positif. Et c'est max(-10, -2 + E(karma/30)) pour un karma négatif ou nul. Avec un karma à 0, ça donne un score de -2. (source)

Pour pouvoir écrire un journal, dans le wiki ou sur la tribune, il faut avoir un karma supérieur strictement à 0 (sources journal, wiki, tribune).

À quoi correspond le rôle sur la page d'un utilisateur ? ¶

Les différents rôles sont :

  • admin : les admins
  • inactive : les comptes fermés
  • moderator : les modérateurs
  • editor : les rédacteurs
  • visitor : les visiteurs, tous ceux qui ne sont pas dans une catégorie précédente.

Quels sont les cookies utilisés par le site ? ¶

Le site utilise actuellement les cookies suivant :

  • remember_account_token
    • durée : courte (deux semaines)
    • usage : cookie de session sur le site
    • concerne : les utilisateurs authentifiés
  • https
    • durée : permanent (jusqu'en 2031)
    • usage : sa présence indiquant que l'ensemble des accès au site doivent se faire en HTTPS
    • concerne : les utilisateurs authentifiés en HTTPS
  • linuxfr.org_session
    • durée : temporaire pour la durée de la session de navigation
    • usage : cookie de session sur le site (utilisable uniquement sur les accès HTTPS si le cookie https précédent existe)
    • concerne : les utilisateurs authentifiés
  • totoz-type
    • durée : cookie permanent (jusqu'en 2042)
    • usage : choisir le type d'affichage des images dans la tribune (en surimpression, insérées dans le texte, et sans images en l'absence du cookie)
    • concerne : les visiteurs de la tribune
